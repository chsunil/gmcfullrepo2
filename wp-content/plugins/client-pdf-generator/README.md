# Client PDF Generator Plugin

## Overview
The Client PDF Generator is a WordPress plugin designed to generate Client and QMS (Quality Management System) PDFs dynamically using AJAX and the DOMPDF library. The plugin integrates seamlessly with WordPress and uses customizable HTML templates to generate PDFs for different certification stages and client data.

## Features
- AJAX-based PDF generation for a smooth user experience.
- Dynamic PDF content rendering using customizable HTML templates.
- Integration with Advanced Custom Fields (ACF) to fetch client-related metadata.
- PDF generation triggered both via frontend AJAX handlers and backend admin POST requests.
- Storage of generated PDFs in the WordPress uploads directory.
- Shortcode support to embed PDF generation functionality in posts or pages.

## Directory Structure
- `client-pdf-generator.php`: Main plugin file that:
  - Loads DOMPDF library.
  - Enqueues required JavaScript.
  - Handles AJAX requests for PDF generation (`cpdf_handle_generate_pdf` function).
- `includes/`: Contains core plugin classes:
  - `class-certification-templates.php`: Handles loading, rendering, and placeholder replacements for PDF templates.
  - `class-dynamic-fields.php`: Fetches client metadata and replaces placeholders in HTML templates.
  - `class-pdf-generator.php`: Encapsulates backend PDF generation and streaming functionality.
- `templates/`: Contains HTML/PHP templates for different client certification stages and schemes.
- `assets/`: JavaScript and other assets used by the plugin.
- `dompdf/`: Bundled DOMPDF library for generating PDFs.

## Main Functions and Classes

### Global Functions
- `cpdf_handle_generate_pdf()`: AJAX handler that validates input, loads templates, generates a PDF via DOMPDF, saves it, updates client post metadata with PDF URL, and returns the URL as AJAX response.

### Classes
- `Certification_Templates`:
  - `init()`: Initializes shortcode for PDF generation.
  - `render($args)`: Renders PDF template with dynamic data for given post ID, type, and stage.
  - `get_template_path($type, $stage)`: Retrieves the absolute path of the template file.
  - `replace_placeholders($html, $data)`: Replaces placeholders in template with actual data.
  - `shortcode_handler($atts)`: Handles shortcode rendering on frontend.

- `Dynamic_Fields`:
  - `get_certification_data($post_id)`: Fetches client metadata such as company name and client type.
  - `replace_placeholders($html, $data)`: Replaces placeholders in HTML content with data values.

- `PDF_Generator`:
  - `init()`: Hooks the backend PDF generation functionality.
  - `generate_pdf()`: Validates request, fetches data, generates PDF, and streams it for download.

## Usage
- Use shortcode `[generate_certification_pdf]` with optional attributes `type`, `stage`, and `post_id` to embed PDF generation in posts/pages.
- AJAX-based PDF generation is invoked via client-side JavaScript enqueued by the plugin.
- Generated PDFs are saved in `wp-content/uploads/client_pdfs/` and linked to client posts for easy access.

## Notes
- The plugin uses DOMPDF, a PHP library to convert HTML to PDF.
- Templates are stored in the `templates/` directory and organized by certification type and stage.
- Custom metadata is retrieved via Advanced Custom Fields or standard WordPress custom fields.
